## Project

**Customer Data Collection** ‚ÄĒ a full-stack web app that collects customer info via a form, applies business logic, and stores records in MySQL. Containerized with Docker Compose; deployable to AWS EC2 via GitHub Actions CI/CD.

## Stack

| Layer | Technology |
|---|---|
| Frontend | HTML5, CSS3, Vanilla JS (Nginx in Docker) |
| Backend | Python 3.11 + Flask |
| Database | MySQL 8.0 |
| Containers | Docker + Docker Compose (v1 ‚ÄĒ `docker-compose` with hyphen) |
| CI/CD | GitHub Actions |

## Commands

```bash
# Run backend tests locally (no Docker / no MySQL needed ‚ÄĒ DB is mocked)
cd backend && pytest tests/ -v

# Start everything (first time or after code changes)
docker-compose up --build

# Start in background
docker-compose up -d

# Stop
docker-compose down

# View logs
docker-compose logs -f backend

# Verify DB records
docker exec -it mysql mysql -u root -prootpassword customerdb -e "SELECT * FROM customers;"

# Build and push images to Docker Hub
docker-compose build
docker push pawarakash2511/customer-api:latest
docker push pawarakash2511/customer-ui:latest
```

## Architecture

```
Browser ‚Üí frontend/:8082 (Nginx) ‚Üí /api/* proxied ‚Üí backend/:5000 (Flask) ‚Üí mysql:3306
```

- **`frontend/nginx.conf`** ‚ÄĒ proxies `/api/` to `http://backend:5000/api/` internally; backend port 5000 is NOT exposed to the host
- **`frontend/script.js`** ‚ÄĒ multiplies `some_number √ó 2` before POSTing to `/api/customers` (relative URL ‚ÄĒ works on localhost and EC2)
- **`frontend/admin.html`** ‚ÄĒ admin panel: login form + customer data dashboard (single page, two states managed by JS)
- **`frontend/admin.js`** ‚ÄĒ handles admin login, JWT storage in `sessionStorage`, data fetching, table rendering
- **`frontend/index.html`** header ‚ÄĒ flexbox row: title (left) + Admin Panel button (right); do NOT use `position:absolute` ‚ÄĒ it caused overlap with the title
- **`backend/app.py`** ‚ÄĒ increments `age + 1`, generates `submitted_at`, calls `insert_customer()`; also exposes `POST /api/admin/login` and `GET /api/admin/customers` (JWT-protected)
- **`backend/database.py`** ‚ÄĒ MySQL connection via env vars (`DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`)
- **`backend/models.py`** ‚ÄĒ `insert_customer(data)` executes the INSERT; `get_all_customers()` fetches all rows for admin panel
- **`database/init.sql`** ‚ÄĒ auto-runs on first MySQL container start; creates `customerdb` and `customers` table

## Admin Panel

- **URL:** `/admin.html` (same port as main app)
- **Access:** "Admin Panel" button in top-right corner of the customer form
- **Auth:** JWT ‚ÄĒ 8-hour session stored in `sessionStorage`; credentials from `ADMIN_USERNAME` / `ADMIN_PASSWORD` env vars (defaults hardcoded in `app.py`)
- **Features:** Table of all submitted customer records (newest first), Refresh button, Logout
- **New dependency:** `PyJWT==2.8.0` in `backend/requirements.txt`
- **New env vars (optional override):** `ADMIN_USERNAME`, `ADMIN_PASSWORD`, `JWT_SECRET`

## Environment

Credentials live in `.env` (gitignored). Copy `.env.example` to `.env` and fill in values before running locally. On EC2, `.env` is auto-generated by the CD workflow from the `DB_PASSWORD` GitHub Secret.

## Responsive Design

Frontend works on desktop, Android, and iPhone. Key techniques in `frontend/style.css` and `frontend/index.html`:
- `viewport-fit=cover` + `env(safe-area-inset-*)` ‚ÄĒ iPhone notch / Dynamic Island
- `font-size: 16px` on all inputs ‚ÄĒ prevents iOS auto-zoom on focus
- `min-height: 44px` on inputs/buttons ‚ÄĒ meets Apple HIG touch target size
- `100dvh` body height ‚ÄĒ fixes iOS Safari address bar issue
- `inputmode="numeric"` on number fields ‚ÄĒ shows numeric keypad on mobile
- `-webkit-text-size-adjust: 100%` ‚ÄĒ prevents font inflation on orientation change

## Tests

`backend/tests/test_app.py` ‚ÄĒ 18 pytest unit tests covering:
- Successful submission (201 + correct field values)
- `age + 1` business logic
- `some_number` stored as received (JS doubles it before POST)
- `submitted_at` is a `datetime` object
- Whitespace stripping on string fields
- All 3 gender options
- Boundary case: age = 0 ‚Üí stored as 1
- Missing / empty field validation (400)
- Invalid JSON, no body (400)
- DB exception ‚Üí 500

## CI/CD (GitHub Actions)

| Workflow | File | Trigger |
|---|---|---|
| CI | `.github/workflows/ci.yml` | Manual (`workflow_dispatch`) |
| CD | `.github/workflows/cd.yml` | Auto, when CI succeeds (`workflow_run`) |

**CI** ‚ÄĒ runs pytest ‚Üí builds backend + frontend Docker images ‚Üí pushes to Docker Hub as `:<sha>` and `:latest`.

**CD** ‚ÄĒ SSHs into AWS EC2 ‚Üí clones repo (first time) or git pulls ‚Üí auto-generates `.env` ‚Üí pulls `:latest` images ‚Üí `docker-compose down && docker-compose up -d` ‚Üí health check ‚Üí prints web URL.

**GitHub Secrets to configure (6 total):**
- `DOCKER_USERNAME` = `pawarakash2511`
- `DOCKER_PASSWORD` = Docker Hub access token
- `EC2_HOST` = EC2 public IP
- `EC2_USER` = `ec2-user`
- `EC2_SSH_KEY` = PEM private key content
- `DB_PASSWORD` = MySQL root password (CD auto-generates `.env` on EC2 from this)

## Port

Frontend is exposed on port `8082` (port 80 was already in use on the EC2 server). AWS Security Group must allow inbound TCP `8082`.
